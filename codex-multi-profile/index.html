Kenapa Perlu

Multi-profile jadi penting saat satu komputer dipakai untuk beberapa identitas Codex: akun personal, akun kerja, akun client, akun tim, atau subscription yang berbeda. Tanpa isolasi, semua login, session, config, dan sebagian integrasi lokal bisa tercampur di satu folder default.

Menghindari salah akun

Task kerja bisa tidak sengaja jalan dari akun personal, atau sebaliknya. Ini rawan untuk billing, audit, dan akses workspace.

Memisahkan quota dan subscription

Kalau tiap akun punya limit atau paket berbeda, profile terpisah membuat pemakaian lebih jelas dan mudah dicek lewat /status.

Menjaga konteks dan riwayat

Session chat, resume context, dan state lokal tidak bercampur antar akun, sehingga lebih mudah tahu agent mana mengerjakan apa.

Mengontrol tool dan akses

Skills, MCP, dan config bisa dibuat berbeda per profile. Akun kerja bisa lebih ketat, akun personal bisa lebih bebas.

Solusinya: buat beberapa folder profile, lalu jalankan Codex dengan CODEX_HOME berbeda untuk tiap akun.

Satu profile Codex = satu folder CODEX_HOME = satu login/session/config lokal.

Konsep CODEX_HOME

CODEX_HOME adalah environment variable yang memberi tahu Codex lokasi folder config dan auth yang harus dipakai.

Command Efek
codex Memakai folder default ~/.codex.
CODEX_HOME="$HOME/.codex-main" codex Memakai folder profile ~/.codex-main.
CODEX_HOME="$HOME/.codex-work" codex Memakai folder profile ~/.codex-work.

Karena profile beda folder, tiap profile bisa punya akun login berbeda.

Contoh Profile

Nama profile bebas. Contoh generic:

Kebutuhan Folder Command pendek
Akun utama/personal ~/.codex-main codex-main
Akun kerja pertama ~/.codex-work-1 codex-work1
Akun kerja kedua ~/.codex-work-2 codex-work2

Kalau ingin nama lain, ganti saja folder dan function. Contoh: ~/.codex-client-a, ~/.codex-team, atau ~/.codex-alt.

Setup Awal

1. Pastikan Codex sudah terinstall

which codex
codex --version
echo "$SHELL"
echo "$HOME"

Kalau codex tidak ditemukan, install Codex dulu sesuai dokumentasi resmi yang berlaku.

2. Backup folder default

cp -R "$HOME/.codex" "$HOME/.codex.backup.$(date +%Y%m%d-%H%M%S)"

Step ini hanya perlu kalau ~/.codex sudah ada.

3. Buat folder profile

mkdir -p "$HOME/.codex-main"
mkdir -p "$HOME/.codex-work-1"
mkdir -p "$HOME/.codex-work-2"

4. Copy config default kalau ada

cp "$HOME/.codex/config.toml" "$HOME/.codex-main/config.toml"
cp "$HOME/.codex/config.toml" "$HOME/.codex-work-1/config.toml"
cp "$HOME/.codex/config.toml" "$HOME/.codex-work-2/config.toml"
Jangan copy auth.json antar profile. File itu berisi data login/token.

Shell Functions

Tambahkan block ini ke ~/.zshrc untuk macOS zsh. Kalau memakai bash, taruh di ~/.bashrc atau ~/.bash_profile.

# >>> codex multi-profile setup >>>
export CODEX_PROJECTS_DIR="$HOME/projects"

codex-main() {
  CODEX_HOME="$HOME/.codex-main" codex "$@"
}

codex-work1() {
  CODEX_HOME="$HOME/.codex-work-1" codex "$@"
}

codex-work2() {
  CODEX_HOME="$HOME/.codex-work-2" codex "$@"
}

codex-main-projects() {
  cd "$CODEX_PROJECTS_DIR" && CODEX_HOME="$HOME/.codex-main" codex "$@"
}

codex-work1-projects() {
  cd "$CODEX_PROJECTS_DIR" && CODEX_HOME="$HOME/.codex-work-1" codex "$@"
}

codex-work2-projects() {
  cd "$CODEX_PROJECTS_DIR" && CODEX_HOME="$HOME/.codex-work-2" codex "$@"
}
# <<< codex multi-profile setup <<<

Sesuaikan CODEX_PROJECTS_DIR dengan folder kerja masing-masing. Contoh: $HOME/projects, $HOME/work, atau $HOME/dev.

Reload shell

source ~/.zshrc

Validasi function

type codex-main
type codex-work1
type codex-work2
type codex-main-projects
type codex-work1-projects
type codex-work2-projects

Skills & MCP

Multi-profile tidak hanya memisahkan login. Ia juga bisa memisahkan config, skills, MCP server settings, dan session lokal, tergantung file apa saja yang ada di masing-masing folder CODEX_HOME.

Item Biasanya ada di Efek multi-profile
Login/token $CODEX_HOME/auth.json Harus berbeda per profile. Jangan dicopy antar profile.
Codex config $CODEX_HOME/config.toml Bisa dicopy kalau ingin setting sama. Review dulu kalau config berisi credential.
Skills $CODEX_HOME/skills/ atau lokasi lain sesuai setup Bisa dipisah per profile, dicopy, atau dibuat shared via symlink.
MCP servers Umumnya di config.toml Profile yang berbeda bisa punya MCP server berbeda.

Opsi 1: Skills dipisah per profile

Pakai opsi ini kalau tiap akun punya workflow berbeda. Misalnya akun personal punya skill eksperimen, akun kerja hanya punya skill yang aman untuk repo kerja.

mkdir -p "$HOME/.codex-main/skills"
mkdir -p "$HOME/.codex-work-1/skills"
mkdir -p "$HOME/.codex-work-2/skills"

Opsi 2: Skills dishare antar profile

Pakai opsi ini kalau ingin semua profile memakai skill yang sama. Simpan skill di satu folder shared, lalu symlink ke tiap CODEX_HOME.

mkdir -p "$HOME/codex-shared-skills"

ln -s "$HOME/codex-shared-skills" "$HOME/.codex-main/skills"
ln -s "$HOME/codex-shared-skills" "$HOME/.codex-work-1/skills"
ln -s "$HOME/codex-shared-skills" "$HOME/.codex-work-2/skills"
Kalau folder skills sudah ada, jangan langsung overwrite dengan symlink. Backup atau merge dulu.

MCP per profile

Jika MCP server dikonfigurasi di config.toml, maka tiap profile bisa punya MCP list sendiri.

# contoh: lihat config profile tertentu
open "$HOME/.codex-main/config.toml"
open "$HOME/.codex-work-1/config.toml"
open "$HOME/.codex-work-2/config.toml"

Kalau ingin semua profile punya MCP sama, copy bagian MCP config ke semua config.toml. Kalau ingin akun kerja lebih terbatas, hapus MCP yang tidak perlu dari profile kerja.

Credential MCP

MCP credential bisa datang dari beberapa tempat: environment variable shell, file credential tool eksternal, browser login, atau isi config. Efeknya beda:

Sumber credential Efek
Environment variable shell Semua profile bisa melihat env yang sama, kecuali function mengatur env berbeda.
Credential di config Ikut kalau config.toml dicopy. Hindari menyimpan secret langsung di config.
Credential tool eksternal Bisa tetap shared di luar CODEX_HOME, tergantung tool MCP-nya.
Auth di dalam CODEX_HOME Terpisah per profile.

Env berbeda per profile

Kalau MCP butuh token berbeda per akun, set env di function profile masing-masing.

codex-main() {
  MCP_API_TOKEN="$MAIN_MCP_API_TOKEN" CODEX_HOME="$HOME/.codex-main" codex "$@"
}

codex-work1() {
  MCP_API_TOKEN="$WORK1_MCP_API_TOKEN" CODEX_HOME="$HOME/.codex-work-1" codex "$@"
}
Jangan taruh token asli langsung di HTML, README publik, atau dotfile yang akan dishare. Gunakan secret manager, env lokal, atau file private yang tidak masuk git.

Cara Pakai

Jalankan dari folder saat ini

codex-main
codex-work1
codex-work2

Jalankan langsung dari folder projects

codex-main-projects
codex-work1-projects
codex-work2-projects

Kirim argumen ke Codex

codex-main -C "$HOME/projects/my-app"
codex-work1 -C "$HOME/projects/my-app"
codex-work2 -C "$HOME/projects/my-app"

Semua argumen setelah function akan diteruskan ke command codex.

Login

Login harus dilakukan per profile. Jalankan satu-satu, lalu pilih akun yang sesuai di browser.

CODEX_HOME="$HOME/.codex-main" codex login
CODEX_HOME="$HOME/.codex-work-1" codex login
CODEX_HOME="$HOME/.codex-work-2" codex login
Selesaikan login satu profile dulu sebelum lanjut ke profile berikutnya. Ini membantu menghindari salah akun saat browser redirect.

Cek /status

Setelah login, buka Codex dari masing-masing profile lalu jalankan /status di dalam Codex.

codex-main
/status
codex-work1
/status
codex-work2
/status

Pastikan akun, workspace, atau subscription yang tampil sesuai profile yang sedang dipakai.

Logout

Logout hanya memengaruhi profile yang dipilih. Profile lain tetap login.

CODEX_HOME="$HOME/.codex-main" codex logout
CODEX_HOME="$HOME/.codex-work-1" codex logout
CODEX_HOME="$HOME/.codex-work-2" codex logout

Login ulang profile tertentu:

CODEX_HOME="$HOME/.codex-work-1" codex login

Session Chat

Session chat terpisah per CODEX_HOME. Jadi session dari codex-work1 tidak otomatis muncul atau bisa dilanjutkan dari codex-work2.

Profile aktif Yang dipakai
codex-main Auth, config, dan session dari ~/.codex-main.
codex-work1 Auth, config, dan session dari ~/.codex-work-1.
codex-work2 Auth, config, dan session dari ~/.codex-work-2.

Kalau ingin lanjut konteks dari profile lain, buat ringkasan manual lalu paste ke session baru. Jangan memindahkan file credential.

Tambah Profile Baru

Misalnya ingin tambah profile ke-4 bernama codex-work3.

1. Buat folder baru

mkdir -p "$HOME/.codex-work-3"

2. Copy config kalau ada

cp "$HOME/.codex/config.toml" "$HOME/.codex-work-3/config.toml"

Lewati step ini kalau ~/.codex/config.toml tidak ada.

3. Tambah function ke file shell config

codex-work3() {
  CODEX_HOME="$HOME/.codex-work-3" codex "$@"
}

codex-work3-projects() {
  cd "$CODEX_PROJECTS_DIR" && CODEX_HOME="$HOME/.codex-work-3" codex "$@"
}

4. Reload shell

source ~/.zshrc

5. Login

CODEX_HOME="$HOME/.codex-work-3" codex login

6. Verify

codex-work3
/status

Kerja di Repo Sama

Boleh menjalankan beberapa profile Codex di repo yang sama, tapi jangan biarkan beberapa agent mengedit file yang sama secara bersamaan.

  • Satu agent fokus satu task.
  • Pisahkan area file yang diedit.
  • Minta agent menyebutkan file yang akan diedit sebelum mulai.
  • Cek git diff sebelum menerima perubahan.
  • Untuk task besar, pakai branch atau git worktree terpisah.

Contoh worktree

cd "$HOME/projects/my-repo"

git worktree add ../my-repo-main -b codex/main
git worktree add ../my-repo-work1 -b codex/work1
git worktree add ../my-repo-work2 -b codex/work2

Security Notes

  • Jangan print isi auth.json.
  • Jangan copy auth.json antar profile.
  • Jangan share API key, password, token, .env, database dump, atau credential kerja.
  • Selalu cek /status sebelum mulai task penting.
  • Kalau akun bukan akun utama, treat sebagai worker account dengan akses seperlunya.

Troubleshooting

Function belum kebaca

source ~/.zshrc
type codex-main
type codex-work1
type codex-work2

Kalau masih belum kebaca, buka terminal baru.

Salah login akun

CODEX_HOME="$HOME/.codex-work-1" codex logout
CODEX_HOME="$HOME/.codex-work-1" codex login

CODEX_HOME kosong saat echo

Normal kalau echo "$CODEX_HOME" kosong di shell biasa. Function di atas memberi CODEX_HOME hanya ke process codex yang sedang dijalankan.

Codex masih pakai akun salah

  1. Exit Codex.
  2. Logout profile yang salah.
  3. Login ulang profile itu.
  4. Buka profile itu lagi.
  5. Jalankan /status.
