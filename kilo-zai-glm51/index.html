Kenapa Setup Ini

Kilo CLI berguna kalau kamu ingin coding agent langsung dari terminal, bukan dari IDE. Dengan Z.AI GLM-5.1, setup ini cocok untuk task coding yang butuh planning panjang, tool use, dan iterasi beberapa langkah.

Intinya: Kilo jadi shell-native coding agent, Z.AI menyediakan model glm-5.1, dan workflow terminal-first bisa dipakai tanpa pindah ke editor plugin.

Install Kilo CLI

Install Kilo CLI via npm. Pastikan Node.js dan npm sudah tersedia dulu.

node --version
npm --version

Install package global:

npm install -g @kilocode/cli

Verify:

which kilo
kilo --version
kilo --help

Start Kilo dari folder project:

cd ~/projects/my-app
kilo

Z.AI API Key

Login ke Z.AI, subscribe atau aktifkan GLM Coding Plan sesuai kebutuhan, lalu buat API key dari halaman API management.

  1. Buka Z.AI Open Platform.
  2. Masuk ke API Keys.
  3. Create API key.
  4. Simpan key di tempat private.

Untuk GLM Coding Plan, gunakan endpoint Coding API:

https://api.z.ai/api/coding/paas/v4

Interactive Setup

Cara paling aman adalah memakai setup interaktif Kilo, karena credential akan dikelola lewat flow CLI.

kilo

Di dalam Kilo, jalankan:

/connect

Pilih provider Z.AI jika tersedia, lalu isi:

  • Base URL: https://api.z.ai/api/coding/paas/v4
  • API key: Z.AI API key kamu
  • Model: glm-5.1
Kalau provider/model belum muncul di UI Kilo, pakai manual config di section berikutnya.

Manual Config

Config global Kilo CLI bisa diletakkan di ~/.config/kilo/opencode.json. Contoh minimal berikut memakai provider Z.AI lewat adapter OpenAI-compatible.

{
  "$schema": "https://opencode.ai/config.json",
  "default_agent": "AGENTS",
  "model": "zai/glm-5.1",
  "provider": {
    "zai": {
      "name": "Z.AI",
      "npm": "@ai-sdk/openai-compatible",
      "options": {
        "baseURL": "https://api.z.ai/api/coding/paas/v4",
        "apiKey": "your_zai_api_key"
      },
      "models": {
        "glm-5.1": {
          "name": "GLM-5.1",
          "reasoning": true,
          "limit": {
            "context": 204800,
            "output": 131072
          }
        }
      }
    }
  }
}

Setelah edit config, restart Kilo:

kilo
Beberapa versi Kilo juga menyebut opencode.jsonc atau kilo.jsonc. Untuk Kilo CLI setup yang diuji di guide ini, path yang dipakai adalah ~/.config/kilo/opencode.json.

MCP & Skills

Kilo bisa memakai MCP dan skills untuk memperluas kemampuan agent. Contoh setup yang diuji memakai MCP seperti context7, playwright, dan postgres.

Folder skills umum:

~/.config/kilo/skills/

Contoh struktur:

~/.config/kilo/
├── opencode.json
└── skills/
    ├── caveman/
    └── commit-hashmicro/
Untuk public guide, cukup jelaskan pattern. Jangan publish skill private yang berisi instruksi internal, credential, nama client, atau workflow sensitif.

Verify

Cek command dan versi:

which kilo
kilo --version
kilo models zai

Smoke test dari folder project:

cd ~/projects/my-app
kilo run "Explain this repository structure briefly."

Atau buka TUI:

kilo

Di dalam Kilo, cek provider/model aktif dari UI atau command status/profile yang tersedia di versi Kilo kamu.

Daily Usage

Mulai Kilo dari root repo supaya agent membaca instruksi project dan struktur file dengan benar.

cd ~/projects/my-app
kilo

Untuk one-shot task:

kilo run "Find the likely cause of this error and propose a minimal fix."

Untuk lanjut session terakhir:

kilo --continue

Untuk session tertentu:

kilo --session <session-id>

Security

  • Simpan API key di config lokal atau credential flow Kilo.
  • Pakai placeholder saat menulis contoh config untuk dokumentasi.
  • Pisahkan config personal, kerja, dan client kalau aksesnya berbeda.
  • Review permission Kilo sebelum memberi auto-approve untuk edit atau shell command.

Troubleshooting

kilo command tidak ditemukan

npm install -g @kilocode/cli
npm bin -g

Pastikan global npm bin ada di PATH.

Model glm-5.1 tidak muncul

Update Kilo, lalu cek config provider manual.

kilo upgrade
kilo models zai

Request masuk endpoint salah

Pastikan base URL memakai Coding API, bukan General API.

https://api.z.ai/api/coding/paas/v4

Auth gagal

Generate API key baru dari Z.AI, update config, lalu restart Kilo. Jangan paste key ke terminal history kalau kamu tidak yakin shell history aman.
